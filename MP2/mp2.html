Issue Deep Dive

For sketch 05, I utilized an object called "sound loops" in p5. It intakes a function and a frequency. Once the sound loops starts, it will run the function every loop. I was able to get the sound loop working - I could input a frequency and add a tone. I could then input another frequency and tones will start playing at each frequency. However, my program was supposed to iterate through the scale and play the next tone every time the user adds a tone while at the same time playing all previous tones. For example, if I add a tone and give it a frequency of 1 second, it should start by playing the C note every 1 second. Then if I add a second tone at a frequency of 2 seconds, it should start playing the D note every 2 seconds while also playing the C note every 1 second. The issue that I faced was that every time I added a tone, all tones will end up playing the same note. In terms of the example I gave, that means that the program would play D every 1 second and D every 2 seconds.

To handle which note the program is supposed to currently be playing, I had a variable named 'currNote' that tracked the current note of the scale. And once it reaches the end of the scale, it should loop back to the beginning. I created a function that the each sound loop I created would take in. All this function did was play the current note each time any sound loop iterated. What I realized was that the sound loop was calling the function every time it looped, which also meant that as I incremented 'currNote,' EVERY SINGLE sound loop I had created would also play that current note. To fix this, I created separate functions for each note. Basically, I have 8 functions and all any of them do is play a different note of the scale. Then, whenever I added a tone and created a sound loop, each sound loop would be fed a different function depending on the current note.
